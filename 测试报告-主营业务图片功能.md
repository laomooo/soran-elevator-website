## 2. 功能实现说明

### 2.1 数据库变更
- 新增 `business_services` 表：
  - `id`（UUID 主键）
  - `title`（业务标题）
  - `description`（业务描述）
  - `image_url`（业务图片 URL）
  - `sort_order`（排序）
  - `is_active`（是否启用）
  - `created_at` / `updated_at`
- 已预置 3 条主营业务数据：
  1. 主流品牌零部件供应
  2. 自主研发生产
  3. 技术服务支持
- RLS 策略：匿名用户可读，认证用户可写
- Storage：新增/复用 `company-assets` bucket，图片存放路径 `business-services/`

### 2.2 后台管理
- 新增 `/admin/services` 页面
- 支持：新增业务、编辑业务、删除业务、调整排序、启用/停用、上传业务图片
- 图片通过 `ImageUpload` 组件上传至 Supabase Storage `company-assets/business-services/`
- 后台侧边栏新增「主营业务」入口

### 2.3 前台展示
- 关于我们页面主营业务改为客户端组件 `BusinessServices`
- 页面加载后自动从 Supabase 拉取最新 `business_services` 数据
- 有图片则显示图片，无图片则显示默认星形占位图标
- 未配置 Supabase 时使用兜底静态数据展示

## 4. 发现问题

### 4.1 主要问题：数据库表未创建
- **现象**：测试脚本读取 `business_services` 时报错 `Could not find the table 'public.business_services' in the schema cache`
- **原因**：本次新增的表尚未在 Supabase 中执行
- **影响**：前台始终显示兜底静态数据，后台无法保存/读取业务数据，图片无法持久化
- **解决方案**：在 Supabase SQL Editor 中执行 `supabase/migration-add-business-services.sql`

### 4.2 已知限制
- 新增业务（产生新的 slug）后，静态站点需要重新构建才能生成对应详情页；本次业务数据为卡片展示，无需生成新路由，刷新页面即可生效
- 后台页面为客户端组件，未登录时仅显示登录表单，静态 HTML 中不包含业务管理 UI（符合预期）

## 5. 部署与使用步骤

### 5.1 执行数据库迁移
1. 登录 Supabase 项目：https://zlrstlwpjhzzixicnqxh.supabase.co
2. 进入 **SQL Editor → New query**
3. 打开项目文件 `soran-website/supabase/migration-add-business-services.sql`
4. 全选复制粘贴到 SQL Editor
5. 点击 **Run**
6. 到 **Table Editor → business_services** 确认 3 条预置数据已生成

### 5.2 验证图片功能
1. 访问 http://localhost:3002/admin/services
2. 用管理员账号登录
3. 点击某条业务的「编辑」
4. 在「业务图片」处上传图片
5. 点击「保存」
6. 访问 http://localhost:3002/about 刷新页面，查看该业务卡片是否显示图片

### 5.3 重新部署（GitHub Pages）
若线上已部署，执行 migration 后需要重新推送代码触发 GitHub Actions，或在 Actions 页面手动触发构建，使静态页面更新。
