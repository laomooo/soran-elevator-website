## 4. 发现问题

### 4.1 中风险：Next.js 14.2.5 安全漏洞
- **描述**：当前 `package.json` 使用 `next@14.2.5`，该版本存在已知安全漏洞（CVE）。
- **建议**：升级到 `next@14.2.15` 或更高 patched 版本。
- **操作**：`npm install next@14.2.15` 后重新构建测试。

### 4.2 低风险：删除资源时未清理 Storage 图片
- **描述**：后台删除产品/分类/新闻/主营业务时，数据库记录被删除，但 Supabase Storage 中的图片文件仍保留。
- **影响**：长期可能产生垃圾文件，占用存储空间。
- **建议**：在删除逻辑中增加 `supabase.storage.from(bucket).remove([path])` 调用。

### 4.3 低风险：Navbar 存在未使用变量
- **描述**：`Navbar.jsx` 中 `isLight` 始终为 `true`，保留了大量 `isLight ? ... : ...` 分支，代码冗余。
- **建议**：清理为固定浅色背景样式，移除未使用分支。

### 4.4 低风险：解决方案详情页缺失
- **描述**：首页解决方案卡片链接统一指向 `/solutions`，没有独立的解决方案详情页。
- **影响**：用户体验可进一步提升，但不影响当前上线。
- **建议**：后续可增加 `/solutions/[slug]` 详情页。

### 4.5 低风险：Storage bucket 列表为空待确认
- **描述**：测试脚本调用 `listBuckets()` 返回空数组。
- **建议**：在 Supabase 控制台确认 `product-images`、`company-assets` 等 bucket 是否已创建；若未创建，执行 `schema.sql` 或手动创建。
