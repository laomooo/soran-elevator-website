## 四、数据库结构

| 表名 | 说明 | 公开访问 |
|------|------|----------|
| `site_config` | 站点配置（单行） | 公开可读 |
| `categories` | 产品分类 | 仅激活分类公开可读 |
| `products` | 产品信息 | 仅激活产品公开可读 |
| `banners` | 首页轮播 | 仅激活Banner公开可读 |
| `news` | 新闻动态 | 仅发布新闻公开可读 |
| `partners` | 合作伙伴 | 公开可读 |
| `admins` | 管理员（关联Auth） | 仅本人可读 |
| `messages` | 用户留言 | 公开可写，仅管理员可读 |

**安全机制**：所有表启用 RLS（Row Level Security），公开用户只能读取，管理员通过 `is_admin()` 函数验证后可执行写入/更新/删除操作。

## 七、后台管理操作指南

### 登录

1. 打开 `https://laomooo.github.io/soran-elevator-website/admin/login.html`
2. 输入 Supabase Auth 中创建的管理员邮箱和密码
3. 登录后进入仪表盘

### 添加产品

1. 点击左侧「产品管理」
2. 点击「添加产品」按钮
3. 填写：型号、名称(中/英)、分类、简短描述、详细描述、规格参数(JSON)
4. 上传产品图片（拖拽或点击上传区域）
5. 设置是否精选、是否上架
6. 点击保存

### 管理轮播图

1. 点击「轮播管理」
2. 添加/编辑Banner：标题、副标题、上传图片、链接地址
3. 拖动调整排序
4. 首页自动展示激活的轮播图

### 站点设置

1. 点击「站点设置」
2. 修改：公司名称、联系方式、地址、关于我们文案
3. 保存后前台即时生效

## 八、本地开发

```bash
# 克隆仓库
git clone https://github.com/laomooo/soran-elevator-website.git
cd soran-elevator-website

# 配置Supabase
# 编辑 docs/js/supabase-client.js 填入你的Supabase URL和anon key

# 本地预览（任意静态服务器）
python3 -m http.server 8000 --directory docs

# 或使用原有Express服务器（需要Node.js）
NODE_OPTIONS="" npm install
npm start
```

## 九、部署流程

1. 所有静态文件在 `docs/` 目录
2. GitHub仓库 Settings → Pages → Source: `Deploy from a branch` → Branch: `master` / `docs`

## 十、常见问题

**Q: 网站打开后显示空白？**
A: 检查 Supabase 配置是否正确（`docs/js/supabase-client.js`），以及 schema.sql 是否已执行。

**Q: 后台登录失败？**
A: 确认已在 Supabase Auth 中创建用户，并在 `admins` 表中插入了对应记录。

**Q: 上传图片失败？**
A: 确认 Supabase Storage 中 `images` 存储桶已创建且权限策略已设置。

**Q: 产品列表不显示？**
A: 检查产品 `is_active` 字段是否为 1，以及分类的 `is_active` 是否为 1。
